## ADDED Requirements

### Requirement: Insight Overview Display
The system SHALL display comprehensive insight configuration and status in the overview tab.

#### Scenario: Configuration summary
- **WHEN** user views insight detail page
- **THEN** system displays insight name, status, created date, last run, next run, connectors, metrics, and AI model

#### Scenario: Status badge
- **WHEN** insight is enabled
- **THEN** system displays green "Enabled" badge in header

#### Scenario: Quick actions
- **WHEN** user views detail page
- **THEN** system displays Edit, Run Now, and actions menu buttons

### Requirement: Recent Reports Section
The system SHALL display 3-5 most recent reports generated by this insight.

#### Scenario: Reports list
- **WHEN** insight has generated reports
- **THEN** system displays recent reports with title, date, format, size, and action buttons

#### Scenario: Empty reports state
- **WHEN** insight has not been run yet
- **THEN** system displays "Insight created, not yet run" with prominent "Run Now" button

#### Scenario: View all reports link
- **WHEN** user clicks "View All Reports"
- **THEN** system navigates to reports tab or `/dashboard/insights/:id/reports`

### Requirement: AI-Generated Insights Display
The system SHALL display AI-generated analysis from the most recent report run.

#### Scenario: Performance summary
- **WHEN** AI analysis is available
- **THEN** system displays performance summary section with key metrics

#### Scenario: Key findings
- **WHEN** AI has identified key findings
- **THEN** system displays findings as bullet points

#### Scenario: Recommendations
- **WHEN** AI has generated recommendations
- **THEN** system displays numbered recommendations

### Requirement: Tab Navigation
The system SHALL provide tabbed navigation for Overview, Reports, Settings, and History.

#### Scenario: Tab switching
- **WHEN** user clicks on a tab
- **THEN** system displays corresponding content panel and updates URL

#### Scenario: Default tab
- **WHEN** user navigates to insight detail
- **THEN** system displays Overview tab by default

### Requirement: Running State
The system SHALL display progress indicator when insight is executing.

#### Scenario: Run now initiation
- **WHEN** user clicks "Run Now"
- **THEN** system makes API call and displays spinner with "Running insight..." message

#### Scenario: Run completion
- **WHEN** insight execution completes
- **THEN** system displays success toast and refreshes data

#### Scenario: Run failure
- **WHEN** insight execution fails
- **THEN** system displays error message and "Retry" button
