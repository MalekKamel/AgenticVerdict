## Context

The authentication module at `/apps/frontend/src/routes/$locale/auth` is currently in its initial implementation phase.

## Objective

Review the planned architecture and requirements documented in:

- `/docs/architecture/ui/04-pages/authentication.md`
- `/docs/architecture/ui/00-overview.md`

Then produce a complete implementation roadmap to bring the current authentication features into full alignment with the documented plan.

## Scope of Work

1. Analyze the current implementation in `/apps/frontend/src/routes/$locale/auth`.
2. Analyze the target plan and requirements in the two documentation files listed above.
3. Perform a gap analysis between implemented behavior and planned behavior.
4. Define a phased implementation roadmap that resolves all identified gaps.

## Deliverables

Provide one or more roadmap files that include:

- A clear inventory of alignment gaps (feature, UX, architecture, routing, state, validation, i18n, accessibility, and security where applicable).
- Prioritized implementation phases with dependencies and sequencing.
- Task-level actions for each phase (what to change, where to change it, and expected outcome).
- Acceptance criteria for each phase that verify 100% alignment with the plan.
- Risks, assumptions, and validation approach (tests, manual checks, and rollout considerations).

## Quality Bar

The final roadmap must be comprehensive, actionable, and unambiguous, with full traceability from each roadmap item to the referenced plan documents, ensuring complete alignment with the intended architecture.
