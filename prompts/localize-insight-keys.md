# Localize Dashboard Insight Keys

## Context

Dashboard insight notifications are constructed in `/apps/api/src/trpc/routers/dashboard.ts` using `titleKey` and `bodyKey` properties. These keys serve as identifiers for localized message content displayed in the UI.

## Objective

Ensure all insight notification keys support complete localization across all format variations, enabling proper Arabic and English message rendering in the user interface.

## Task

1. **Audit**: Analyze `/apps/api/src/trpc/routers/dashboard.ts` to identify all `titleKey` and `bodyKey` construction patterns and their possible values
2. **Catalog**: Document all unique key formats and variations generated by the dashboard router
3. **Localize**: Add corresponding translations to the i18n system (`packages/i18n/`) for both Arabic and English
4. **Verify**: Confirm all keys render correctly in the UI with proper RTL/LTR text direction

## Acceptance Criteria

- All insight title and body keys have Arabic and English translations defined
- Keys follow the existing i18n naming conventions
- UI displays localized messages correctly for both languages
- No hardcoded strings remain in the dashboard router

## Files to Review

- `/apps/api/src/trpc/routers/dashboard.ts` - Key construction logic
- `/packages/i18n/` - Translation definitions
- `/apps/frontend/` - UI components consuming insight notifications
