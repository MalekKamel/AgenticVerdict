# Insights & Reports Routes Documentation

## Overview

This document describes the routing structure for the Insights and Reports features in the AgenticVerdict frontend application.

## Route Structure

### Insights Routes

```
/en/dashboard/insights              # Insight list page
/en/dashboard/insights/new          # Create insight wizard
/en/dashboard/insights/:id          # Insight detail page
/en/dashboard/insights/:id/edit     # Edit insight wizard
```

### Reports Routes

```
/en/dashboard/reports               # Report list page
/en/dashboard/reports/:id           # Report viewer page
```

### Shared/Public Routes

```
/en/shared/reports/:id              # Shared report access (public, requires token)
```

## Route Details

### Insight List Page

**Path:** `/dashboard/insights`

**Features:**

- List all insights with pagination
- Filter by status (enabled/disabled/all)
- Search by name
- Action menu (view, edit, run now, delete)
- "New Insight" button to create wizard

**Query Parameters:**

- `status` - Filter by insight status
- `search` - Search term
- `page` - Page number
- `pageSize` - Items per page

**Component:** `apps/frontend/src/features/insights/pages/InsightListPage.tsx`

### Insight Create Wizard

**Path:** `/dashboard/insights/new`

**Features:**

- 6-step wizard for creating insights
- Step 1: Basic Information (name, description, domain)
- Step 2: Connector Selection
- Step 3: Metric Configuration
- Step 4: AI Settings (model, quality, detail level)
- Step 5: Schedule & Delivery (frequency, recipients)
- Step 6: Review & Create

**Component:** `apps/frontend/src/features/insights/pages/InsightCreateWizard.tsx`

**Navigation:**

- On success: Navigate to `/dashboard/insights/:id`
- On cancel: Navigate to `/dashboard/insights`

### Insight Detail Page

**Path:** `/dashboard/insights/:id`

**Features:**

- Tabbed navigation (Overview, Reports, Settings, History)
- Configuration summary
- Recent reports section
- AI-generated insights card
- "Run Now" action
- Status badge

**Tabs:**

1. **Overview** - Configuration summary and recent reports
2. **Reports** - List of reports generated by this insight
3. **Settings** - Edit insight configuration
4. **History** - Audit trail timeline

**Component:** `apps/frontend/src/features/insights/pages/InsightDetailPage.tsx`

### Insight Edit Page

**Path:** `/dashboard/insights/:id/edit`

**Features:**

- Same 6-step wizard as create
- Pre-populated with current values
- Dirty state detection
- "Unsaved changes" indicator
- "Reset to Default" for each section

**Component:** `apps/frontend/src/features/insights/pages/InsightEditPage.tsx`

**Navigation:**

- On success: Navigate to `/dashboard/insights/:id`
- On cancel: Navigate to `/dashboard/insights/:id`

### Report List Page

**Path:** `/dashboard/reports`

**Features:**

- Table view with sortable columns
- Filter by date range, format, status
- Search by name
- Bulk selection and actions (download, delete)
- Row actions (view, download, share, delete)
- Pagination

**Query Parameters:**

- `dateFrom` - Start date for filter
- `dateTo` - End date for filter
- `format` - Filter by format (pdf/excel/all)
- `status` - Filter by status (ready/generating/failed)
- `search` - Search term
- `page` - Page number
- `pageSize` - Items per page

**Component:** `apps/frontend/src/features/reports/pages/ReportListPage.tsx`

### Report Viewer Page

**Path:** `/dashboard/reports/:id`

**Features:**

- PDF viewer with zoom controls (50% - 200%)
- PDF page navigation
- Excel preview with sheet tabs
- Actions: print, download, share
- Version selector for reports with multiple versions

**Components:**

- `apps/frontend/src/features/reports/pages/ReportViewerPage.tsx`
- `apps/frontend/src/features/reports/ui/ReportViewer.tsx`
- `apps/frontend/src/features/reports/ui/ExcelViewer.tsx`

### Shared Report Access

**Path:** `/shared/reports/:id`

**Query Parameters:**

- `token` - Required share token for access

**Features:**

- Public route (no authentication required)
- View-only access to shared report
- Download action available
- Handles expired/invalid tokens

**Component:** `apps/frontend/src/features/reports/pages/SharedReportPage.tsx`

## Navigation Examples

### Programmatic Navigation

```typescript
import { useRouter } from "@/router";

function MyComponent() {
  const router = useRouter();

  // Navigate to insights list
  router.push("/dashboard/insights");

  // Navigate to create wizard
  router.push("/dashboard/insights/new");

  // Navigate to insight detail
  router.push(`/dashboard/insights/${insightId}`);

  // Navigate to insight edit
  router.push(`/dashboard/insights/${insightId}/edit`);

  // Navigate to reports list
  router.push("/dashboard/reports");

  // Navigate to report viewer
  router.push(`/dashboard/reports/${reportId}`);

  // Navigate to shared report
  router.push(`/shared/reports/${reportId}?token=${shareToken}`);
}
```

### Using Link Component

```typescript
import { Link } from "@/router";

function MyComponent() {
  return (
    <>
      <Link href="/dashboard/insights">View Insights</Link>
      <Link href="/dashboard/insights/new">Create New Insight</Link>
      <Link href={`/dashboard/insights/${insightId}`}>View Details</Link>
      <Link href={`/dashboard/insights/${insightId}/edit`}>Edit Insight</Link>
      <Link href="/dashboard/reports">View Reports</Link>
      <Link href={`/dashboard/reports/${reportId}`}>View Report</Link>
    </>
  );
}
```

### With Query Parameters

```typescript
import { useRouter } from "@/router";

function MyComponent() {
  const router = useRouter();

  // Filter insights by status
  router.push("/dashboard/insights?status=enabled");

  // Search insights
  router.push("/dashboard/insights?search=SEO");

  // Filter reports by date range
  router.push("/dashboard/reports?dateFrom=2024-01-01&dateTo=2024-12-31");

  // Filter reports by format
  router.push("/dashboard/reports?format=pdf");

  // Combined filters
  router.push("/dashboard/reports?status=ready&format=pdf&page=2");
}
```

## Route Guards

### Authentication Required

All insights and reports routes require authentication:

- `/dashboard/insights/*`
- `/dashboard/reports/*`

### Public Routes

The following routes are public (no auth required):

- `/shared/reports/*` (requires valid token)

### Tenant Context

All routes require tenant context from the JWT. The tenant ID is automatically extracted and used in all API calls.

## Feature Flags

The insights and reports UI is controlled by the `ENABLE_INSIGHTS_UI` feature flag.

```typescript
import { isFeatureEnabled } from "@/lib/feature-flags";

if (isFeatureEnabled("ENABLE_INSIGHTS_UI")) {
  // Show insights/reports navigation
}
```

## Error Handling

### Route Not Found

If an insight or report doesn't exist, the application will:

1. Show a 404 error page
2. Provide navigation back to the list

### API Errors

API errors are handled by the canonical error system:

1. Display user-friendly error message
2. Provide retry option
3. Log error with tenant context

### Shared Report Errors

For shared reports:

- **Invalid token**: Show "Invalid share link" message
- **Expired token**: Show "Share link expired" message
- **Revoked token**: Show "Share link no longer available" message

## Localization

All routes support localization:

- English: `/en/dashboard/insights`
- Arabic: `/ar/dashboard/insights`

The locale is automatically prefixed based on user preference.
