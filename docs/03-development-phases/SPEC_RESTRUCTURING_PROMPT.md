## `/speckit-specify`

Build a restructured documentation system for the AgenticVerdict development phases that reflects the platform's evolution from a marketing analytics agent into a **multi-business-domain intelligence platform**.

**Why this restructuring is needed:**

The AgenticVerdict platform has expanded beyond its original scope as a marketing analytics tool for Masafh (B2B GPS fleet tracking). It now supports multiple business domains:

- **Marketing**: Meta, GA4, GSC, GBP, TikTok analytics
- **Finance**: Planned QuickBooks, Stripe integration
- **Operations**: Extensible connector framework
- **Agency**: Multi-tenant architecture for partner organizations

The current phase structure (`phase-00` through `phase-03`) was designed for the original marketing-focused architecture and no longer reflects:

1. The multi-business-domain nature of the platform
2. The tight coupling between data connectors and intelligence generation
3. The business architecture's emphasis on insights as the primary deliverable
4. The completion status of foundational components

**What needs to be built:**

A consolidated documentation structure under `/specs/` with:

1. **`00-core/`** — A unified specification consolidating foundational phases into logical sub-phases:
   - `00-foundation`: Infrastructure, multi-tenancy, core services (from `phase-00-foundation`)
   - `01-connectors`: Data connector integrations and normalization (from `phase-01-platform-integration`)
   - `02-intelligence`: AI agent orchestration and analysis (from `phase-02-agent-intelligence`)
   - `03-insights`: Report generation and delivery (from `phase-03-report-generation`)

2. **Updated index** — Root `README.md` reflecting the new structure with clear navigation

**Key requirements:**

- **Legacy documentation REMOVAL:** All existing documentation at `/docs/03-development-phases/` must be **removed** after successful migration to `/specs/` (not preserved as historical reference)
- New consolidated specifications will be created in `/specs/`
- All internal references between documents must be updated to point to `/specs/`
- **ALL codebase references must be updated** — including comments, documentation strings, README files, and any references in source code files
- Cross-references from other documentation sections must be updated to `/specs/`
- CLAUDE.md development phase references must be updated to `/specs/`
- The `/docs/03-development-phases/` directory must be **completely removed** after migration is complete
- The structure must accommodate future domains and features without restructuring

Each SpecKit-generated specification must include:

1. **Overview** — Definition and relationship to business architecture
2. **Business Domains** — Connector-to-domain mapping and cross-domain capabilities
3. **Phase Dependencies** — Visual dependency graph with blocking relationships
4. **Acceptance Criteria Summary** — Aggregated criteria with status tracking

4. **Codebase-wide reference update:**
   - Search all `.md` files for references to old phase paths
