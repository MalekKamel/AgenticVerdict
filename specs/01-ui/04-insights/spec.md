## User Scenarios & Testing _(mandatory)_

### User Story 1 - Configure and Activate Insight (Priority: P1) ⭐

As a business user, I need to create and configure insights by selecting templates, connectors, and metrics so that I can track the business metrics that matter most to my organization and receive AI-generated analysis.

**Why this priority**: This is the PRIMARY VALUE FEATURE of the platform. Insights are the core product - users create insights to automatically analyze their business data across connectors. Without this feature, the platform cannot deliver its primary value proposition of AI-powered business intelligence.

**Independent Test**: Can be fully tested by creating a complete insight through the wizard: selecting a template, choosing connectors, picking metrics, configuring AI settings, and activating the insight. The deliverable is a functional insight that appears in the insight list and can generate reports.

**Acceptance Scenarios**:

1. **Given** a user accesses the insight creation wizard, **When** they select a template (e.g., "Marketing Performance"), **Then** the wizard pre-fills recommended connectors and metrics for that template
2. **Given** a user is on the connector selection step, **When** they choose a connector (e.g., "GA4"), **Then** the wizard displays available metrics for that connector with multi-select capability
3. **Given** a user has selected connectors and metrics, **When** they proceed to the AI configuration step, **Then** they can choose model (Claude 3.5 Sonnet / GPT-4o), quality level (fast/balanced/thorough), and detail level (concise/standard/comprehensive)
4. **Given** a user completes the wizard with all required selections, **When** they click "Activate Insight", **Then** the insight is saved, appears in the insight list with "Active" status, and generates its first report
5. **Given** a user with existing insights, **When** they clone an insight, **Then** a new insight is created with identical configuration that can be independently modified

---

### User Story 2 - View Insight Feed and Take Action (Priority: P2) ⭐

As a business user, I need to view a feed of AI-generated insights with data visualizations so that I can quickly understand performance trends, identify anomalies, and take informed actions.

**Why this priority**: This is the consumption side of the PRIMARY VALUE FEATURE. Users need to consume the insights generated by the platform. While P1 focuses on creating insights, P2 focuses on deriving value from them through an intuitive feed interface.

**Independent Test**: Can be fully tested by generating insights for active configurations and viewing them in the feed interface. The deliverable is a scrollable feed of insight cards with charts, key metrics, and AI recommendations.

**Acceptance Scenarios**:

1. **Given** a user with active insights, **When** they navigate to the insight feed, **Then** they see a chronological feed of insights with the most recent first
2. **Given** an insight card in the feed, **When** the user views it, **Then** the card displays key metrics, trend charts, AI-generated summary, and actionable recommendations
3. **Given** an insight showing declining performance, **When** the user clicks "View Details", **Then** they navigate to the insight detail page with full analysis and drill-down capabilities
4. **Given** an insight with recommendations, **When** the user clicks "Take Action" on a recommendation, **Then** the system provides context-relevant actions (e.g., "Create Campaign", "Adjust Budget", "View Connector")
5. **Given** a user viewing the feed, **When** they use filters (date range, connectors, insight type), **Then** the feed updates to show only matching insights

---

### User Story 3 - Manage and Monitor Insights (Priority: P1)

As a business user, I need to view, search, filter, and manage all my insights so that I can monitor their status, edit configurations, and troubleshoot issues.

**Why this priority**: This is foundational to the insight management experience. Users need to see all their insights in one place, understand their status (active/inactive/error), and take management actions. Without this, users cannot effectively oversee their intelligence operations.

**Independent Test**: Can be fully tested by creating multiple insights with different statuses, then using the insight list interface to search, filter, and manage them. The deliverable is a comprehensive insight management interface.

**Acceptance Scenarios**:

1. **Given** a user with multiple insights, **When** they navigate to the insight list page, **Then** they see all insights with name, status (active/inactive/error), last run, next run, and quick actions
2. **Given** a user on the insight list, **When** they use the search box to search by name, **Then** the list filters to show only matching insights
3. **Given** a user on the insight list, **When** they apply filters (status, connector, date range), **Then** the list updates to show only insights matching the filter criteria
4. **Given** a user viewing an insight in the list, **When** they click the edit action, **Then** they navigate to the insight edit interface with all current configurations pre-loaded
5. **Given** an insight with error status, **When** the user views it, **Then** they see error details and suggested remediation steps

---

### User Story 4 - Analyze Insight Details with Data Visualization (Priority: P2)

As a business user, I need to view detailed analysis with interactive data visualizations so that I can deeply understand performance trends, compare metrics over time, and validate AI recommendations.

**Why this priority**: This provides the detailed analysis experience for power users. While the feed gives quick overviews (P2 US2), the detail view enables deep exploration with full charts, tables, and comprehensive AI analysis.

**Independent Test**: Can be fully tested by opening an insight detail view and interacting with all visualization components (charts, tables, filters, drill-downs). The deliverable is a comprehensive detail interface with multi-dimensional analysis.

**Acceptance Scenarios**:

1. **Given** a user viewing an insight detail page, **When** the page loads, **Then** they see key metrics at top, trend charts in middle, and AI analysis below
2. **Given** trend charts on the detail page, **When** a user hovers over data points, **Then** they see tooltips with exact values and timestamps
3. **Given** a user viewing charts, **When** they adjust the date range picker, **Then** all charts update to show data for the selected period
4. **Given** a user analyzing multiple metrics, **When** they toggle metrics on/off in the chart legend, **Then** the chart updates to show only selected metrics
5. **Given** a user reviewing AI recommendations, **When** they click "View Full Analysis", **Then** they see the comprehensive AI-generated report with methodology, data quality indicators, and detailed findings

---

### User Story 5 - Modify Existing Insight Configuration (Priority: P2)

As a business user, I need to edit existing insight configurations so that I can adapt to changing business needs, add new metrics, or refine AI parameters based on results.

**Why this priority**: Insights need to evolve as business needs change. While creation is critical (P1 US1), the ability to modify ensures insights remain relevant over time without requiring deletion and recreation.

**Independent Test**: Can be fully tested by editing an active insight: changing connectors, adding/removing metrics, adjusting AI settings, and saving changes. The deliverable is an updated insight that reflects the new configuration.

**Acceptance Scenarios**:

1. **Given** a user editing an insight, **When** they access the edit interface, **Then** all current configurations are pre-loaded in the wizard steps
2. **Given** a user on the connector selection step, **When** they add a new connector, **Then** the system shows available metrics for that connector without affecting existing selections
3. **Given** a user on the metric selection step, **When** they remove a metric, **Then** the system warns if this is the last metric for a connector
4. **Given** a user on the AI configuration step, **When** they change quality level from "fast" to "thorough", **Then** the system updates estimated processing time and cost
5. **Given** a user saves edits to an active insight, **When** the save completes, **Then** the insight continues running with the new configuration on the next scheduled run

### Edge Cases

- What happens when a user tries to create an insight without selecting any connectors?
- How does the system handle connector disconnection after insight creation?
- What occurs when a connector API rate limit is reached during insight generation?
- How does the wizard behave when a selected connector has no available metrics?
- What happens when a user edits an active insight and removes all metrics?
- How does the system handle very large metric datasets that exceed chart rendering capacity?
- What occurs when an insight generation fails due to AI service unavailability?
- How does the feed behave when there are hundreds of insights across multiple months?
- What happens when a user clones an insight and the original is deleted?
- How does the system handle concurrent edits to the same insight by multiple users?
- What occurs when a template used by an insight is deleted or modified?
- How does the detail view handle missing or corrupted historical data?
- What happens when date range selection spans periods with no data?
- How does the system handle insights with connectors from different business domains (Marketing + Finance)?
- What occurs when a user's permissions change and they lose access to a connector used by an insight?

## Requirements _(mandatory)_

### Functional Requirements

#### Insight List Page

- **FR-001**: System MUST display all insights in a table/card format with name, description, status, last run time, next run time, and quick actions
- **FR-002**: System MUST provide search functionality to filter insights by name or description
- **FR-003**: System MUST provide filter controls for status (active/inactive/error), connector type, date range, and business domain
- **FR-004**: System MUST support sorting insights by name, status, last run, next run, and creation date
- **FR-005**: System MUST provide quick actions: view, edit, clone, activate/deactivate, delete
- **FR-006**: System MUST display visual status indicators (green for active, gray for inactive, red for error, yellow for pending)
- **FR-007**: System MUST show insight count summary by status (e.g., "5 Active, 2 Inactive, 1 Error")
- **FR-008**: System MUST support pagination or virtual scrolling for large insight lists (50+ insights)

#### Insight Creation Wizard

- **FR-009**: System MUST provide a multi-step wizard with progress indicator showing current step
- **FR-010**: System MUST include wizard steps: Template Selection → Connector Selection → Metric Selection → AI Configuration → Schedule & Delivery → Review & Activate
- **FR-011**: System MUST allow users to skip template selection and start from scratch
- **FR-012**: System MUST support "Back" and "Next" navigation between wizard steps
- **FR-013**: System MUST save draft state automatically so users can exit and resume later
- **FR-014**: System MUST validate each step before allowing progression to the next step
- **FR-015**: System MUST provide a summary review step showing all selections before activation
- **FR-016**: System MUST allow users to jump to any previously completed step via progress indicator

#### Template Selection Step

- **FR-017**: System MUST display available templates in a card/grid format with name, description, and preview
- **FR-018**: System MUST show recommended connectors and metrics for each template
- **FR-019**: System MUST support template preview showing sample insight output
- **FR-020**: System MUST filter templates by business domain (Marketing, Finance, Operations, SEO, Social, Local)
- **FR-021**: System MUST allow users to select "Custom" to build an insight without a template

#### Connector Selection Step

- **FR-022**: System MUST display all available connectors for the tenant with connection status (connected/not connected)
- **FR-023**: System MUST show connector health indicators (healthy/degraded/error)
- **FR-024**: System MUST allow multi-select of connectors
- **FR-025**: System MUST display connector type icons and names (e.g., "GA4", "Meta", "GSC")
- **FR-026**: System MUST warn if a selected connector is not connected and provide "Connect" action
- **FR-027**: System MUST show connector business domain tags
- **FR-028**: System MUST recommend connectors based on template selection (if template selected)

#### Metric Selection Step

- **FR-029**: System MUST display available metrics grouped by connector
- **FR-030**: System MUST show metric name, description, data type, and granularity
- **FR-031**: System MUST support multi-select of metrics within and across connectors
- **FR-032**: System MUST require at least one metric to be selected before proceeding
- **FR-033**: System MUST display recommended metrics based on template selection
- **FR-034**: System MUST show metric data availability indicators (e.g., "30 days history available")
- **FR-035**: System MUST support metric search and filtering
- **FR-036**: System MUST warn if a connector has no metrics selected and suggest removal

#### AI Configuration Step

- **FR-037**: System MUST provide model selection (Claude 3.5 Sonnet, GPT-4o, etc.)
- **FR-038**: System MUST provide quality level selection (Fast, Balanced, Thorough) with descriptions of trade-offs
- **FR-039**: System MUST provide detail level selection (Concise, Standard, Comprehensive) affecting output length
- **FR-040**: System MUST display estimated processing time and cost for selected AI configuration
- **FR-041**: System MUST allow users to set custom AI prompts or instructions
- **FR-042**: System MUST provide default AI prompts per template
- **FR-043**: System MUST support advanced AI settings (temperature, max tokens) for power users

#### Schedule & Delivery Step

- **FR-044**: System MUST provide schedule options: On-demand, Daily, Weekly, Monthly
- **FR-045**: System MUST allow custom cron expression input for advanced scheduling
- **FR-046**: System MUST display next run time based on schedule selection
- **FR-047**: System MUST provide time zone selection (default to tenant time zone)
- **FR-048**: System MUST offer delivery options: Email, In-App Feed, Webhook (future)
- **FR-049**: System MUST support multiple email recipients with validation
- **FR-050**: System MUST allow email subject and message customization
- **FR-051**: System MUST provide delivery format options (PDF, HTML, Excel)

#### Insight Detail View

- **FR-052**: System MUST display insight header with name, description, status, and last run time
- **FR-053**: System MUST show key metrics in summary cards at top (e.g., "Total Sessions", "Conversion Rate", "Revenue")
- **FR-054**: System MUST provide interactive trend charts using Recharts with line/area/bar visualizations
- **FR-055**: System MUST support date range selection (Last 7 days, Last 30 days, Custom)
- **FR-056**: System MUST allow metric toggle in charts to show/hide specific metrics
- **FR-057**: System MUST display AI-generated summary and recommendations prominently
- **FR-058**: System MUST provide "View Full Report" action to open complete PDF/HTML report
- **FR-059**: System MUST show data quality indicators (completeness, freshness, anomalies)
- **FR-060**: System MUST support metric drill-down to view underlying data tables
- **FR-061**: System MUST provide export functionality for charts (PNG, CSV)
- **FR-062**: System MUST display insight configuration summary (connectors used, metrics tracked, AI settings)

#### Insight Edit Interface

- **FR-063**: System MUST provide edit interface using the same wizard flow as creation
- **FR-064**: System MUST pre-load all existing configurations in the wizard
- **FR-065**: System MUST show which step each configuration belongs to
- **FR-066**: System MUST allow users to modify any step of the configuration
- **FR-067**: System MUST warn if changes will significantly impact insight output
- **FR-068**: System MUST validate changes before saving
- **FR-069**: System MUST provide "Save Changes" and "Cancel" actions
- **FR-070**: System MUST preserve insight activation status (active insights remain active after edit)
- **FR-071**: System MUST show version history of insight configurations

#### Insight Clone Functionality

- **FR-072**: System MUST provide "Clone" action on insight list and detail views
- **FR-073**: System MUST create a new insight with identical configuration to the original
- **FR-074**: System MUST append "(Copy)" to the cloned insight name
- **FR-075**: System MUST navigate to the edit interface for the cloned insight after creation
- **FR-076**: System MUST set cloned insight status to "Inactive" by default
- **FR-077**: System MUST preserve all selections: template, connectors, metrics, AI config, schedule, delivery

#### Insight Feed (P2)

- **FR-078**: System MUST display insights in reverse chronological feed format
- **FR-079**: System MUST show insight cards with key metrics, sparkline charts, and summary
- **FR-080**: System MUST support infinite scroll for large feed volumes
- **FR-081**: System MUST provide feed filters: date range, connectors, insight type, status
- **FR-082**: System MUST allow users to mark insights as "Read" or "Important"
- **FR-083**: System MUST provide "Load More" button at feed bottom
- **FR-084**: System MUST update feed in real-time or on refresh when new insights generate
- **FR-085**: System MUST show unread count badge on feed navigation
- **FR-086**: System MUST support feed refresh via pull-to-refresh or refresh button

#### AI Configuration Panel

- **FR-087**: System MUST provide collapsible AI configuration panel within wizard
- **FR-088**: System MUST display current AI model, quality level, and detail level
- **FR-089**: System MUST show estimated cost per insight generation
- **FR-090**: System MUST provide advanced settings toggle for temperature, max tokens, top-p
- **FR-091**: System MUST allow custom system prompt configuration
- **FR-092**: System MUST validate custom prompts for length and content
- **FR-093**: System MUST provide prompt templates for common use cases
- **FR-094**: System MUST show AI usage statistics (tokens used, cost this month)

#### Error Handling & Validation

- **FR-095**: System MUST display user-friendly error messages for all failure scenarios
- **FR-096**: System MUST provide inline validation for all form inputs
- **FR-097**: System MUST warn users before destructive actions (deletion, deactivation)
- **FR-098**: System MUST show loading states for all async operations
- **FR-099**: System MUST provide retry functionality for failed insight generations
- **FR-100**: System MUST log all errors for debugging and support

#### Accessibility Requirements

- **FR-101**: All wizard steps MUST be keyboard navigable with proper focus management
- **FR-102**: All charts MUST include alternative text and data table representations
- **FR-103**: All form inputs MUST have associated labels visible to screen readers
- **FR-104**: Color MUST NOT be the only means of conveying status (use icons + text)
- **FR-105**: All interactive elements MUST have visible focus indicators
- **FR-106**: System MUST support browser and OS high contrast mode
- **FR-107**: All modals and overlays MUST trap focus and support Escape to close
- **FR-108**: Tooltips MUST be keyboard accessible and screen reader compatible

#### Internationalization Requirements

- **FR-109**: All UI strings MUST be externalized to translation files
- **FR-110**: Date and time formats MUST respect locale settings
- **FR-111**: Number and currency formats MUST respect locale settings
- **FR-112**: Chart labels and axes MUST support RTL text direction
- **FR-113**: Wizard layout MUST mirror correctly for RTL languages
- **FR-114**: System MUST preserve text direction in mixed-language content

#### Performance Requirements

- **FR-115**: Insight list page MUST load within 2 seconds on 3G connection
- **FR-116**: Wizard step transitions MUST complete within 500ms
- **FR-117**: Insight detail page MUST render charts within 1.5 seconds
- **FR-118**: Feed MUST display initial 10 insights within 2 seconds
- **FR-119**: Chart interactions (hover, toggle) MUST respond within 100ms
- **FR-120**: System MUST cache connector and metric data to avoid repeated API calls

### Key Entities

- **Insight**: A configurable business intelligence entity that defines which connectors to use, which metrics to track, AI analysis parameters, schedule, and delivery settings. Insights generate reports on a recurring schedule.
- **Template**: A pre-built configuration that accelerates insight creation by providing recommended connectors, metrics, AI prompts, and settings for specific use cases (e.g., "Marketing Performance", "Finance Overview").
- **Connector**: A data integration connection to an external platform (e.g., GA4, Meta, GSC) that provides metrics for insight generation. Connectors have health status and authentication state.
- **Metric**: A specific data point tracked from a connector (e.g., "Sessions", "Conversions", "Revenue"). Metrics have data types, granularity, and availability ranges.
- **InsightFeedItem**: A generated insight instance displayed in the feed with timestamp, key metrics, trend summary, AI recommendations, and read status.
- **AIConfiguration**: Settings for AI analysis including model selection (Claude, GPT-4), quality level (fast/balanced/thorough), detail level (concise/standard/comprehensive), custom prompts, and advanced parameters.
- **ScheduleConfiguration**: Recurring schedule settings for insight generation including frequency (daily/weekly/monthly), time, time zone, and custom cron expressions.
- **DeliveryConfiguration**: Settings for insight delivery including channels (email, in-app, webhook), recipients, format preferences (PDF/HTML/Excel), and message customization.

## Assumptions

- Phase 03 (Connectors) is complete, providing connector list, health status, and metric discovery APIs
- TanStack Start and Mantine v9 are configured from Phase 00 (Foundation)
- Dashboard layout and navigation exist from Phase 02 (Scaffold)
- tRPC routers for insights CRUD operations are implemented or will be implemented in parallel
- Report generation infrastructure from Phase 02 (Intelligence) can generate reports on demand
- AI agent runtime from Phase 02 can analyze metrics and generate insights
- Users have appropriate permissions to create, edit, and delete insights
- Connector authentication and OAuth flows are handled by the connector management system
- Insight scheduling will be handled by BullMQ background jobs
- Email delivery will use Resend/SendGrid integration (to be implemented)
- Chart visualizations will use Recharts library with RTL support
- State management for wizard will use TanStack Store
- Form validation will use Zod schemas
- Primary target browsers are modern evergreen browsers (Chrome, Firefox, Safari, Edge)
- Mobile responsiveness is required but desktop is the primary interface
- Arabic and English are the primary languages requiring RTL/LTR support
- Insight generation may take 10-60 seconds depending on AI configuration and data volume
- Cost tracking for AI usage will be displayed but not enforced (no budgets)
- Template system will be basic for Phase 04 (P1) with enhanced template management in Phase 06
